# RUDN-Webdev-HW2

React-приложение Kanban-доски для управления задачами. Реализовано в рамках домашнего задания по курсу "Промышленная веб-разработка".

## О проекте

Приложение представляет собой упрощенную версию Kanban-доски для отслеживания задач по их статусу выполнения. Пользователь может создавать задачи, изменять их статус, просматривать детальную информацию и удалять задачи.

Демо: https://sagderovkamalit.github.io/RUDN-Webdev-HW2/

## Основные возможности

- Просмотр задач в трех колонках по статусам: "К выполнению", "В работе", "Выполнено"
- Создание новых задач с указанием названия и описания
- Просмотр детальной информации о задаче
- Изменение статуса задачи (перемещение между колонками)
- Удаление задач
- Автоматическое сохранение данных в localStorage браузера
- Навигация между страницами через React Router

## Используемые технологии

### Frontend Framework
- **React 19** - библиотека для построения пользовательского интерфейса
- **TypeScript** - типизированный JavaScript для повышения надежности кода

### Сборка и инструменты
- **Vite** - быстрый сборщик для разработки и production сборки
- **ESLint** - линтер для проверки качества кода

### UI библиотека
- **Material UI (MUI)** - библиотека компонентов для создания современного интерфейса
- **@mui/icons-material** - набор иконок Material Design

### Маршрутизация
- **React Router** - библиотека для навигации между страницами в SPA приложении

### Работа с данными
- **Tanstack Query (React Query)** - библиотека для управления серверным состоянием, кеширования и синхронизации данных
- **Axios** - HTTP клиент для выполнения запросов к API

### Хранение данных
- **localStorage** - локальное хранилище браузера для сохранения задач

## Установка и запуск

### Требования
- Node.js версии 18 или выше
- npm или yarn

### Установка зависимостей
```
npm install
```

### Запуск в режиме разработки
```
npm run dev
```
Приложение будет доступно по адресу http://localhost:5173

### Сборка для production
```
npm run build
```
Собранные файлы будут находиться в папке `dist`

### Предпросмотр production сборки
```
npm run preview
```

### Деплой на GitHub Pages
```
npm run deploy
```
Команда автоматически соберет проект и задеплоит его в ветку `gh-pages`


## Страницы приложения

### Главная страница (/board)
Отображает три колонки с задачами, отсортированными по статусам:
- К выполнению (статус 0)
- В работе (статус 1)
- Выполнено (статус 2)

На этой странице можно только просматривать задачи и переходить к детальной информации, кликнув по карточке задачи.

### Создание задачи (/create)
Форма для создания новой задачи. Пользователь указывает:
- Название (обязательное поле)
- Описание (необязательное поле)

При создании автоматически присваиваются номер, дата создания и начальный статус "К выполнению".

### Просмотр задачи (/task/:id)
Страница с детальной информацией о задаче:
- Номер задачи
- Название
- Описание
- Дата создания
- Текущий статус

На этой странице можно изменить статус задачи или удалить ее.

## Работа с данными

Начальные данные загружаются с JSONPlaceholder API (endpoint `/todos`). Формат данных с API преобразуется в формат приложения:
- `completed: false` → статус 0 (К выполнению)
- `completed: true` → статус 2 (Выполнено)
- Добавляются поля `description` (null) и `createdAt` (текущая дата)

Все изменения (создание, изменение статуса, удаление) сохраняются в localStorage браузера и сохраняются между сессиями.

## Особенности реализации

- Использование кастомных хуков для работы с данными через Tanstack Query
- Кеширование данных для оптимизации производительности
- Типизация всех компонентов и функций через TypeScript
- Адаптивный дизайн с использованием Material UI
- Обработка состояний загрузки и ошибок
- Валидация форм при создании задач
